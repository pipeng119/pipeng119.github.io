序号 分类 命令 描述 备注
1 git branch（分支操作） git branch <new branch> 新建分支 不切换
2 git branch 查看本地所有分支
3 git branch -r 查看远程所有分支
4 git branch -a 查看本地和远程的所有分支
5 git branch -m <new branch> 修改当前分支名
6 git branch -m <local branch > <new branch> 修改不是当前分支的分支名
7 git branch -d <local branch> 删除本地分支(即将要删除的分支，已经合并到当前分支)
8 git branch -D <local branch> 删除本地分支(即将要删除的分支，没有合并到当前分支)
9 git push origin branch --delete <remote branch> 删除远程分支
10 git branch -v 查看本地各分支最后的提交信息
11 git branch --merged 查看哪些分支已经合并入当前分支
12 git checkout （用于切换分支活恢复工作书文件） git checkout <local branch> 切换本地分支
13 git checkout -b <new branch> 新建分支并切换到该分支
14 git checkout -b <new branch> origin/<remote branch> 把远程分支拉取到本地，作为新的分支
15 git checout . 把当前目录所有修改的文件恢复成未修改时的状态
16 git checkout --<file name> 恢复当前目录下的某个文件
17 git fetch git fetch 查看远程版本库中的更新，取到本地
18 git fetch -p 查看远程已删除的分支
19 git tag(标签操作) git tag 查看远程所有tag
20 git tag ### 创建本地tag
21 git push --tag 创建本地tag
22 git tag -d ### 删除本地tag
23 git push origin tag --delete ### 删除远程tag
24 文件变更、保存操作 git add . 跟踪本地发生变化的文件
25 git status | git status -a 查看文件处于什么状态
26 git diff -cached 比较缓存区与最新本地版本库
27 git diff 比较工作区与缓存区的差异
28 git pull 从远程拉取最新代码
29 git push 把该分支提交到远程
30 git merge（合并分支） git merge <local branch> 把本地某分支合并到该分支
31 git merge --abort 取消合并
32 git cherry-pick <commit_id> 将某分支的commit_id提交合并到当前分支
33 git remote (远程仓库操作) git remote 查看关联的远程仓库的名称
34 git remote -v 查看关联的远程仓库的地址
35 git remote add repositiry_name repository_url 新增远程仓库
36 git remote remove repositiry_name 删除远程仓库
37 git remote set-url <new-repositiry-name> <new-repository-url> 直接修改远程仓库名称和地址
38 git reset (撤销commit操作) git reset --soft HEAD^ 只撤销commit记录，所有改动都保留（工作区和暂存区） HEAD 最近一个提交
HEAD^1上一次提交
git add . 后改动的文件在暂存区(stage diretory),未add之前改动的文件在工作区(wording diretory)
39 git reset --mixed HEAD^ 撤销commit记录，所有改动都保留在工作区
39 git reset --hard HEAD^ 不仅是撤销commit记录，所有改动都被撤销(工作区和暂存区)
40 git rev-parse HEAD 获取本地commit后的commit id
41 git diff HEAD HEAD^ --name-only 获取本次修改/增删的文件列表
